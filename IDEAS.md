# MaxNow 想法记录

这个文件用来长期保存 MaxNow 的产品想法、未来入口、暂时搁置的方向和待研究问题，避免每次聊完之后想法散掉。

## 使用规则

- 新想法先记在这里，之后再判断要不要进入实现。
- 可以保留原始想法，也可以补一段简短解释。
- 每个想法标记状态：`active`、`parked`、`research`、`done`。
- 这里只代表“已记录的想法”，不等于已经进入当前版本范围；真正确定要做的内容再同步进 `SPEC.md`。

## 正在研究

### Last-30 滚动记忆

- 日期：2026-06-14
- 状态：research
- 来源：Owner 想法

给 MaxNow 增加一层“最近 30 天”的滚动上下文，让它每天自动维护今天、本周和近 30 天的大事。

目标：

- 每天保留今日大事，而不是只看当下状态。
- 每周沉淀本周推进、完成、变化和卡点。
- 最近 30 天保留持续主线、重要决定和等待项。
- 让 MaxNow 逐渐从看板变成个人连续记忆。

初步方向：

- 单独维护 `data/last-30.json` 和 `data/last-30.js`。
- 单独维护 `openclaw/last-30/SKILL.md`。
- Home 可以先显示紧凑摘要，完整详情以后再决定是否需要新页面。
- 自动更新由服务器上的执行器触发，优先复用 OpenClaw 的定时任务边界。

待判断问题：

- Last-30 是否独立成数据文件，还是先嵌入 `data/dashboard.json`？
- 每天新增事实来自 Git、OpenClaw 运行结果、Token 统计、手动记录，还是以后接更多来源？
- 哪些判断必须标记为 `needsOwnerConfirm`，避免自动化替 Owner 下结论？

### 桌面伴随面板

- 日期：2026-06-12
- 状态：research
- 来源：Owner 想法

给 MaxNow 增加桌面级入口，让个人状态工作站不需要每次打开浏览器也能被看到。

macOS 方向：

- 做一个顶部状态栏里的小应用。
- 点击状态栏图标后，下拉一个紧凑的个人面板。
- 面板显示今日状态、当前主线、今日推进、关键时间点和简洁的系统状态。
- 气质应该像私人控制台，不像通知流。

Windows 方向：

- 做一个桌面壁纸式个人看板。
- 它更像覆盖在桌面上的平静状态层，而不是普通应用窗口。
- 显示和网页看板相同的核心状态信息，但适配常驻、扫一眼就能懂的场景。
- 不要变成完整 todo 工具，也不要变成新闻墙。

共同原则：

- 浏览器里的 `dash.maxnow.cn` 仍然是 v1 的标准入口。
- 桌面入口尽量复用同一套数据契约。
- 个人状态优先，外部 AI 输入保持次要。
- 交互保持紧凑、安静、偏操作型。

待判断问题：

- 桌面伴随面板应该是独立应用、静态网页外壳，还是直接读取 `data/dashboard.json` 的小原生应用？
- Windows 版本应该是真壁纸、透明置底窗口，还是桌面 widget 层？
- macOS 和 Windows 后续要共用一个跨平台框架，还是分别用平台原生方案？
