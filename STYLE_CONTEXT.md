# MaxNow 前端样式上下文

这个文件记录 Dash 和 Blog 的视觉约定。以后调整页面样式前先读这里，避免把已经确认过的视觉方向回退掉。

## 总体气质

- MaxNow 是私人状态工作站和公开笔记发布层，不做营销页气质。
- 界面要平静、紧凑、可扫读，优先服务每天反复打开时的效率。
- 当前视觉基准是轻量用户中心风格：白色 sidebar、浅蓝灰背景、柔和白卡片、淡边框、克制阴影、彩色小图标入口和系统字体。
- 不要回退到深色重阴影、巨大 hero、装饰性渐变球、外部 Web Font 或过度插画化背景。

## 品牌区

- MaxNow 正式图标是 Owner 在 2026-06-21 确认的深蓝 `M/N` 标识。
- Dash 图标资产：`dash/assets/maxnow-icon.png`。
- Blog 图标资产：`blog/assets/maxnow-icon.png`。
- 左侧品牌区使用 28px 图标，无底框，只显示 `MaxNow`。
- Blog 品牌区不要显示 `blog.maxnow.cn` 副标题；Dash 和 Blog 对齐。
- 不要回退到旧的浅蓝 `M` SVG，也不要给 Blog 图标加蓝色底框。

## 圆角和卡片

- Dash / Blog / Blog Preview 的页卡圆角基准是 `14px`。
- 导航小图标容器圆角基准是 `12px`。
- 主要页卡包括：Home 状态卡、panel、Blog article card、Blog topic card、豆奶顶部 tab、豆奶图表卡。
- 页卡内部的小信息块通常优先继承 `var(--radius)`，但豆奶指标例外：豆奶指标放在外层 tab / panel 内时不要再单独画边框或白底，避免形成“小框套小框”。
- pill 标签使用 `999px` 圆角，例如状态标签、日期标签、单位角标。
- 不要把卡片套卡片做成厚重层级；重复项、工具面板、小指标块可以用卡片，页面大区块保持轻量。

## 悬浮交互

- 桌面端可 hover 的页卡需要有轻微悬浮感。
- 当前标准：边框颜色变为语义色或浅蓝、阴影增强、`transform: translateY(-2px)`。
- 豆奶顶部三个 tab 也要保持 hover 悬浮，不要只让内部小指标悬浮。
- 触摸端不要强行依赖 hover 效果；响应式布局首先保证不挤出、不遮挡。
- Home 豆奶摘要卡作为可点击入口，应保持整体 hover / focus 可感知。

## 颜色使用

- 不要让页面变成单一蓝色系。蓝色是主色，但状态和资源指标需要多元语义色。
- 当前语义色：
  - 蓝：主信息、流量、默认强调。
  - 橙：到期、推进、需要注意但不是错误。
  - 青：日均、时长、辅助资源。
  - 紫：累计、Token、计数。
  - 绿：健康、正常、累计延长。
  - 红：错误、失败、危险状态。
- Home 状态卡应保持多色区分：当前主线蓝、今日推进橙、Token 紫、数据健康绿。
- 豆奶顶部参数固定为：
  - 剩余流量：蓝
  - VIP 到期：橙
  - 日均可用：青
  - 累计签到：紫
  - 累计流量：蓝
  - 累计延长：绿
- 豆奶顶部参数要保留小 SVG 图标和对应语义色，不要回退到纯文字参数块或全蓝 / 全青。
- 豆奶指标的结构固定为：左侧小图标 + 右侧标题同一行，数值放在下一行；图标和数值使用同一个语义色。

## Home 小日历

- Home 顶部右侧的时间卡是紧凑小日历，不只是时钟。
- 固定展示公历日期、当前时间、农历日期和当天节日。
- 节日提示保持轻量，使用橙色强调；没有节日时显示弱提示，不要扩大成单独卡片。

## 豆奶展示

- 豆奶详情页顶部拆成三个并列 tab：标题、账号余量、签到累计。
- 三个 tab 在桌面端并排，窄屏下堆叠，不能横向溢出。
- 豆奶详情页顶部参数使用紧凑单位：
  - 天数使用 `d`，例如 `55d`。
  - 小时使用 `h`，例如 `1.50h`。
  - 天 + 小时使用 `3d 16h`。
  - 每日单位使用 `/d`，例如 `4.2 GB/d`。
- 图表单位角标使用 `GB/d`、`MB/d`、`h/d`。
- Home 豆奶摘要固定两行三列：
  - 第一排：今日流量、今日豆丁、今日延长。
  - 第二排：累计签到、累计流量、累计延长。
- Home 豆奶摘要同样使用“icon + 标题 / 数值”结构，不要给每个指标再套独立小框。
- Home 只放豆奶摘要入口，不放趋势图；趋势图属于豆奶详情页。

## 图表和宽度

- 折线图容器要铺满卡片内部宽度。
- 图表可以保留最小渲染宽度以保证刻度可读，但容器不能让页面横向溢出。
- 先检查 `scrollWidth > clientWidth`，再确认是否是图表内部横向滚动，而不是整页溢出。
- 图表标题保留中文说明，单位角标可以用紧凑英文单位。

## Blog 对齐

- Blog 的 sidebar、品牌区、卡片圆角、hover 和图标容器要和 Dash 对齐。
- Blog 是公开笔记发布层，仍然保持工具型、归档型气质，不做营销 landing。
- 文章卡、专题卡、总览卡都使用同一圆角和悬浮规则。
- Blog 顶部入口和 Dash 顶部入口保持弱按钮样式，不抢主内容注意力。

## 修改样式时的检查

- 改 Dash 样式时提升 `dash/index.html` 中的 `styles.css?v=...`。
- 改 Dash 脚本渲染时提升 `dash/index.html` 中的 `app.js?v=...`。
- 改 Blog 样式时提升所有 Blog 页面里的 `styles.css?v=...`。
- 改 Blog Preview 样式时提升 `preview.css?v=...`。
- 完成后运行 `python scripts/check.py`。
- 对 UI 改动至少验证：
  - 首页没有横向溢出。
  - 豆奶页没有横向溢出。
  - Blog 首页没有横向溢出。
  - hover 卡片没有改变布局尺寸。
  - 长中文、日期、`d/h` 单位不会挤出小卡。
